2. 博文编写

因为用的是markdown语言，所以关于格式啊语法啊啥的还是有点讲究的。这里偶然间花线了一个在线的Markdown编写器，直接对照着现学，灰常之快~另外它还有基本和高阶的Markdown语法。这东西应该是越用越熟吧。

另外当你撸完了这些语法后发现为什么还是有诸如段首空格以及空行之类的问题的话，请不要麻烦地Google答案去吧（好欠扁的指导╭(╯^╰)╮）。不过谷完了要记住就是了~像我这等早期健忘痴呆患者的话就得及时记下来！ 　　 以下就是给我等重度健忘者的备忘：

  • 首先关于Markdown的段首缩进问题：

    　　Markdown 语言不负责实现段首缩进 (ˊ_>ˋ)
    　　段首缩进这件事，应该是 CSS 或者其他排版工具的事情，Markdown 奉行的是样式和内容分开的哲学。
    　　即使手动输入空格，&nbsp 也是很不推荐的方法。我推荐全角空格，切换到全角模式下（一般的中文输入法都是按 shift + space）输入两个空格就行了。这个相对 &nbsp 来说稍微干净一点，而且宽度是整整两个汉字，很整齐。
    　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　——摘自知乎

  • 然后是空行问题：

　由于Markdown语法支持直接使用HTML注记，所以直接用html的空行方法< br/ >就行了~

　另外，根据 Markdown 语法（Daring Fireball: Markdown Syntax Documentation），当需要插入换行符时，可在一行的结尾加上两个或更多的空格。但是据说Mou的预览不支持这种方法。直接用第一种好了嘛~

3.1 执行git命令时，操作失败：permission denied（publick key) 这是我在将source提交到仓库的source分支下时出的问题

进行到第三步时的error。

原因：没有设置好ssh keys
解决： 参考官方教程(https://help.github.com/articles/generating-ssh-keys)

这里可能会粗线问题o(╯□╰)o就是

$ ssh-add ~/.ssh/id_rsa

这一步么，可能会有could not open a connection to your authentication agent的错误提示

解决：先执行命令`ssh-agent bash`，再接着往下做。
